# 产品设计文档

## 一、产品定位

### 1.1 产品愿景

Education Agent 是一款面向**小学 → 初中 → 高中 → 大学 → 职业培训 → 成人终身学习**全学段的 AI 智能体助手，通过大语言模型、知识图谱和向量检索等技术，为学生、教师、成人学习者、家长及教学管理者提供智能化教学辅助服务，提升教学效率和学习体验。

### 1.2 目标用户

#### 核心用户

| 用户角色 | 描述 | 核心诉求 |
|----------|------|----------|
| 小学生 (6-12) | 好奇心强，注意力短，需家长监督 | 趣味学习、习惯养成、基础巩固 |
| 初中生 (12-15) | 自主意识增强，科目增多 | 学科提分、中考备考、学习方法 |
| 高中生 (15-18) | 学业压力大，目标明确 | 高考冲刺、薄弱学科突破、选科指导 |
| 大学生 (18-22) | 自主学习，专业分化 | 专业课辅导、考证考研、毕业设计 |
| 成人学习者 | 职业转型/提升/考证/兴趣 | 系统路径、碎片化学习、即学即用 |

#### 服务用户

| 用户角色 | 描述 | 核心诉求 |
|----------|------|----------|
| 小学教师 | 班级管理、基础教学 | 趣味出题、作业批改减负、学情追踪 |
| 中学教师 | 应试教学、升学压力 | 精准出题、成绩分析、薄弱点诊断 |
| 大学教师 | 专业教学、科研兼顾 | 课件生成、论文辅助、研究生指导 |
| 培训机构讲师 | 商业化教学、学员留存 | 个性化教案、学员进度管理、效果展示 |
| 家长 | 关注孩子学习效果 | 学情可见、效果可量化、使用安全 |
| 教学管理者 | 教务人员、课程负责人 | 学情分析、教学质量监控、预警系统 |

### 1.3 产品价值主张

| 对象 | 价值主张 |
|------|----------|
| **对学生** | 7x24 智能辅导，个性化学习路径，精准定位薄弱环节，学段自适应内容呈现 |
| **对教师** | 自动化教学辅助工具（出题/批改/课件），释放重复劳动，专注教学设计 |
| **对家长** | 学情报告实时推送，学习效果可视化，使用时长安全管控 |
| **对管理者** | 数据驱动的教学质量分析，学生预警，区域教育数据对比 |
| **对成人** | 职业导向学习路径，碎片化学习适配，模拟面试/证书备考 |

## 三、核心功能设计

### 3.1 智能对话辅导（全学段）

**场景**: 用户在学习过程中遇到问题，通过对话方式向 AI 助手提问。

**学段自适应策略**:

| 学段 | 内容呈现方式 | 交互风格 |
|------|------------|---------|
| 小学 | 趣味语言 + 图文动画 + 故事类比 | 引导式，多用鼓励 |
| 初中 | 知识点精讲 + 典型例题 | 苏格拉底式追问 |
| 高中 | 考点标注 + 解题思路 | 结构化推理 |
| 大学 | 学术讨论 + 论文引用 | 专业深度对话 |
| 成人 | 实际案例 + 工作场景 | 实用导向 |

**关键设计点**:
- 多轮对话上下文记忆
- 回答引用具体知识点
- 根据用户画像自适应内容深度和呈现方式
- 未成年人对话增加安全护栏

### 3.2 智能出题系统

**场景**: 教师需要生成练习题/试卷，或学生需要针对性练习。

**交互流程**:
1. 教师选择课程/知识点、设定难度、题型、数量
2. AI 根据知识图谱和难度要求生成试题
3. 支持学段课标对齐（人教版/北师大版/苏教版）
4. 教师预览、调整、确认
5. 试题入库，可发布给学生

**关键设计点**:
- 支持单选、多选、判断、填空、简答多种题型
- 自动生成参考答案和评分标准
- 难度自适应（根据学生水平调整）
- 学段课标对齐

### 3.3 作业批改与考试

**场景**: 学生提交作业后，AI 自动批改并给出反馈。

**功能矩阵**:

| 功能 | 适用学段 | 说明 |
|------|---------|------|
| 作业批改 + 反馈 | 全学段 | 客观题自动判分 + 主观题 AI 评分 |
| 错题归因分析 | 全学段 | 定位到具体知识点掌握不足 |
| 中高考模拟考 | 初中、高中 | 全真模拟环境，AI 评分排名 |
| 证书考试刷题 | 大学、职业培训 | PMP/CPA/公考等定制刷题 |
| 考研/考公辅导 | 大学 | 针对性备考 + 真题训练 |
| 考前 AI 押题 | 高中、大学、职业 | 基于知识图谱权重推荐 |

### 3.4 教师助手

**场景**: 教师日常教学工作的 AI 辅助。

| 功能 | 描述 | 适用学段 |
|------|------|---------|
| 智能助教 | 自动回答学生常见问题 | 全学段 |
| 一键课件 | 按学段风格自动生成 PPT 大纲/讲义 | 全学段 |
| 质量分析 | 发现"大多数学生不会的知识点" | 全学段 |
| 班级对比 | 不同班级同一知识点掌握对比 | 小学-大学 |
| 难度预测 | 基于历史数据预测试卷难度 | 初中-大学 |
| 素材库 | AI 搜集整理教学素材 | 全学段 |

### 3.5 学习路径规划

**场景**: 用户需要系统化学习，AI 制定个性化计划。

| 路径类型 | 适用对象 | 特点 |
|---------|---------|------|
| 学期/备考计划 | 中学生 | 对齐课标，按天安排 |
| 专业学习计划 | 大学生 | 按课程体系，含先修关系 |
| 职业转型路径 | 成人 | 从入门到就业，含实战项目 |
| 碎片化学习 | 成人 | 每天 10-15 分钟，通勤适配 |
| 证书备考路径 | 成人/大学生 | 针对特定考试，定制刷题 |

### 3.6 家长端

**场景**: 家长关注孩子的学习情况和效果。

| 功能 | 描述 | 适用学段 |
|------|------|---------|
| 学情日报/周报 | 推送学习时长、掌握变化、薄弱环节 | 小学、初中 |
| 亲子学习 | 家长和孩子一起完成 AI 设计的任务 | 小学 |
| 时长管控 | 设置使用上限，到时提醒锁定 | 小学、初中 |
| 效果可视化 | 图表展示学习进步趋势 | 小学、初中 |

### 3.7 管理后台

**场景**: 教学管理者和教育局的数据分析需求。

| 功能 | 描述 |
|------|------|
| 学情大屏 | 实时展示班级/年级/全校学习热力图 |
| 学生预警 | 自动识别"可能掉队"学生，按学段不同预警维度 |
| 课程评估 | 基于学习数据和反馈评估课程质量 |
| 区域分析 | 教育局视角，区域内学校对比 |
| 教师看板 | 教师使用情况、AI 教学效果 |

### 3.8 游戏化激励

**场景**: 通过游戏化元素激励各学段用户持续学习。

| 元素 | 小学 | 中学 | 大学/成人 |
|------|------|------|---------|
| 成就 | 星星/勋章（视觉丰富） | 段位/等级 | 学习证书 |
| 积分 | 完成练习获取积分 | 积分兑换资料 | 积分兑换功能 |
| 闯关 | 冒险故事关卡 | 知识闯关 | - |
| 排行 | 班级排行 | 年级/跨校 PK | 学习排行榜 |

## 四、非功能性需求

### 4.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 健康检查响应时间 | < 100ms |
| 普通 API 响应时间 | < 500ms |
| AI 对话首 Token 响应 | < 2s |
| 向量检索响应时间 | < 300ms |
| 语音识别响应时间 | < 1s |
| 并发用户支持 | 100+ |

### 4.2 可用性要求

| 指标 | 目标值 |
|------|--------|
| 系统可用率 | 99.5%+ |
| 数据持久性 | 99.9% |
| 故障恢复时间 | < 30min |

### 4.3 安全要求

| 要求 | 说明 | 适用对象 |
|------|------|---------|
| 未成年人内容安全 | AI 输出审核 + 安全护栏 | 小学/初中/高中 |
| 数据隐私保护 | 敏感数据加密 + 个保法合规 | 全学段 |
| API 接口鉴权 | JWT / OAuth2 | 全学段 |
| Prompt 注入防护 | 输入预处理 + 意图检测 | 全学段 |
| 操作日志审计 | 关键操作追溯 | 全学段 |
| 使用时长管控 | 家长设置上限 | 小学/初中 |

### 4.4 可扩展性

- 支持水平扩展（无状态服务设计）
- AI 模型可切换（Spring AI 抽象层 + 多模型路由）
- 插件化工具注册（Function Calling）
- 知识库支持动态扩展
- 学段模块化，可按学段独立部署或组合
