# 前端页面交互设计文档

> **技术栈**: HTML5 + Tailwind CSS + DaisyUI + Alpine.js
>
> **设计规范**: 学段自适应主题 · 响应式布局 · 无障碍访问 · SEO/GEO 优化

---

## 一、设计系统

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| 学段自适应 | 同一页面根据用户学段自动调整 UI 风格（小学→卡通趣味，高中→简洁专业，成人→商务简约） |
| 渐进式呈现 | 核心功能首屏展示，高级功能按需展开，避免信息过载 |
| 即时反馈 | 所有操作提供 Loading 状态、成功/失败 Toast、骨架屏占位 |
| 容错设计 | 输入校验实时提示，危险操作二次确认，支持撤销 |

### 1.2 学段主题风格

| 学段 | 主色调 | 风格 | 字体 | 图标风格 |
|------|--------|------|------|----------|
| 小学 | #FF6B35 暖橙色 | 圆角大、卡片式、插画丰富 | 圆润字体 | 手绘风 |
| 初中 | #4A90D9 天蓝色 | 半圆角、清爽简洁 | 标准字体 | 线性图标 |
| 高中 | #2C5F8A 深蓝色 | 直角、专业紧凑 | 标准字体 | 线性图标 |
| 大学 | #1A365D 藏青色 | 学术风、留白多 | 衬线+无衬线 | 极简图标 |
| 职业培训 | #0D7377 墨绿色 | 商务风、效率导向 | 无衬线 | 商务图标 |
| 成人学习 | #6B46C1 紫色 | 简约、沉浸阅读 | 无衬线 | 极简图标 |

### 1.3 通用组件规范

| 组件 | 规范 |
|------|------|
| 导航栏 | `<header>` 顶部固定，含 Logo + 学段切换器 + 消息通知 + 用户头像下拉菜单 (DaisyUI `navbar` + `dropdown`) |
| 侧边栏 | 可折叠，按角色展示不同菜单项，当前页高亮 (DaisyUI `drawer` + `menu`) |
| 卡片 | 圆角 8-16px（学段自适应），阴影 (DaisyUI `card` 组件) |
| 按钮 | 主按钮(学段色) / 次按钮(描边) / 危险按钮(红色)，尺寸 S/M/L (DaisyUI `btn`) |
| 空状态 | 插画 + 文案 + 引导操作按钮 |
| 加载态 | 骨架屏(DaisyUI `skeleton`) 优先，其次 `loading` spinner |
| 弹窗 | 居中 Modal (DaisyUI `modal`) / 右侧 Drawer (详情查看) / 底部 ActionSheet (移动端) |
| Toast | 成功(绿) / 失败(红) / 警告(黄) / 信息(蓝)，3s 自动消失 (DaisyUI `toast` + `alert`) |
| 表单 | 输入框/选择器/单选等 (DaisyUI `form-control` + `input` + `select` + `radio`) |
| 数据展示 | 表格/统计/排行 (DaisyUI `table` + `stats` + `badge`) |

### 1.4 响应式断点

| 断点 | 宽度 | 布局策略 |
|------|------|----------|
| Mobile | < 768px | 单列，底部 Tab 导航，全屏弹窗 |
| Tablet | 768px - 1024px | 双列，侧边栏可折叠 |
| Desktop | 1024px - 1440px | 多列，固定侧边栏 |
| Wide | > 1440px | 多列，内容区最大宽度 1200px 居中 |

**交互说明**:

| 元素 | 交互 | 状态 |
|------|------|------|
| Tab 切换 | 点击切换登录方式，表单区域平滑切换 | 默认: 密码登录 |
| 用户名输入 | 实时校验格式，失焦提示错误 | 错误: 红色描边 + 提示文案 |
| 密码输入 | 支持显示/隐藏切换(眼睛图标) | 默认: 隐藏 |
| 登录按钮 | 点击后 Loading 态，禁用重复提交 | 加载中: 按钮文字变为"登录中..." |
| 第三方登录 | 点击弹出对应 OAuth 授权窗口 | 点击: 跳转授权页 |
| 注册链接 | 点击跳转 `/register` | - |

**状态处理**:
- **初始态**: 表单空白，登录按钮可用
- **校验失败**: 对应输入框红色描边 + 错误文案，按钮不可点击
- **登录中**: 按钮 Loading，所有输入禁用
- **登录成功**: Toast "登录成功"，跳转首页 (根据角色路由到不同首页)
- **登录失败**: Toast 显示错误信息 (账号不存在/密码错误/账号锁定)

## 七、通用交互规范

### 7.1 全局状态管理

| 状态类型 | 方案 | 说明 |
|---------|------|------|
| 用户信息 | Alpine.js `$store` (全局 store 插件) | 全局存储用户基本信息、学段、角色 |
| 学段主题 | Alpine.js `x-data` + CSS Variables | 学段切换时动态切换 `data-stage` 属性，CSS 变量联动 |
| 对话状态 | Alpine.js 组件级 `x-data` | 管理当前会话、消息列表、流式输出状态 |
| 路由状态 | 服务端渲染 + `<a href>` 原生导航 | MPA 多页架构，服务端路由守卫 + 角色权限控制 |
| 服务端状态 | fetch API + Alpine.js `x-init` | 页面加载时获取数据，结合 `localStorage` 缓存 |
| 表单状态 | 原生 HTML5 校验 + Alpine.js 自定义校验 | 实时校验提示，提交状态管理 |

### 7.2 页面加载策略

| 策略 | 实现 | 说明 |
|------|------|------|
| 服务端渲染 (SSR) | 服务端直出 HTML | MPA 架构，每个页面独立 HTML，搜索引擎可直接抓取 |
| 骨架屏 | DaisyUI `skeleton` 组件 | 数据加载前展示页面结构，提升感知性能 |
| 资源预加载 | `<link rel="prefetch">` + `<link rel="preload">` | 关键资源预加载，菜单悬浮预加载目标页 |
| 接口缓存 | `localStorage` + `Cache-Control` | 静态数据本地缓存 + HTTP 缓存策略 |
| 懒加载 | `loading="lazy"` 原生属性 | 图片、iframe 等原生懒加载支持 |
| 增量加载 | Alpine.js `x-intersect` 指令 | 元素进入视口时触发数据加载 (IntersectionObserver) |

### 7.3 错误处理

| 场景 | 处理方式 |
|------|----------|
| 401 Token 过期 | 自动刷新 Token，失败则跳转登录页 |
| 403 权限不足 | Toast 提示 "权限不足"，不跳转 |
| 404 资源不存在 | 展示空状态页 + 返回首页按钮 |
| 500 服务器错误 | Toast "服务异常，请稍后重试" |
| 网络断开 | 顶部 Banner 提示 "网络已断开"，禁用操作 |
| AI 回复超时 | 30s 超时提示 "AI 响应超时" + 重试按钮 |
| 内容安全拦截 | 消息区显示 "该内容无法展示" + 反馈入口 |

### 7.4 无障碍设计 (A11y)

| 要求 | 实现 |
|------|------|
| 键盘导航 | 所有交互元素支持 Tab 键聚焦，Enter/Space 触发 |
| 屏幕阅读器 | ARIA 标签、角色属性、实时区域 (aria-live) |
| 色彩对比 | WCAG 2.1 AA 标准，对比度 ≥ 4.5:1 |
| 动效减弱 | 尊重 `prefers-reduced-motion` 系统设置 |
| 字体缩放 | 支持浏览器字体缩放至 200% |

### 7.6 离线与弱网支持

| 功能 | 策略 |
|------|------|
| 离线缓存 | Service Worker 缓存静态资源 + 已加载页面 |

#### 9.1.2 页面级 SEO 规范

| 元素 | 规范 | 示例 |
|------|------|------|
| `<title>` | 核心关键词 + 学段 + 品牌名，≤ 60字符 | “二次函数顶点式讲解 - 高中数学 \| 教育培训智能体” |
| `<meta description>` | 150字内，含关键词，描述页面核心价值 | “AI智能讲解高中数学二次函数顶点式...” |

| `<img alt>` | 所有图片含描述性 alt 文本 | alt="二次函数顶点式示意图" |

### 9.2 GEO (AI 生成式搜索优化)

> 针对 Google SGE、Bing Copilot、Perplexity、ChatGPT Search 等 AI 搜索引擎的优化策略。

#### 9.2.1 内容结构化与权威性

| 策略 | 实现 | 说明 |
|------|------|------|
| 清晰的问答结构 | 每个知识点页包含“什么是...”、“如何...”等 H2 标题 | AI 搜索直接引用问答对 |
| 定义式开头 | 每个知识点第一段用简洁语言给出定义 | 便于 AI 抽取作为“答案摘要” |
| 分步讲解 | 解题过程用 `<ol>` 有序列表展示步骤 | AI 倾向引用结构化的步骤内容 |
| 权威引用 | 标注课标来源、教材版本 | 提升内容可信度，AI 优先引用权威来源 |
| 多角度解释 | 同一知识点提供定义、公式、图形、实例多角度 | 增加被 AI 搜索引用的覆盖面 |

#### 9.2.2 实体化内容组织

```html
<!-- 每个知识点作为独立实体，具备完整信息 -->
<article itemscope itemtype="https://schema.org/LearningResource">
  <h1 itemprop="name">勾股定理</h1>

  <!-- 定义 (AI 搜索最爱引用) -->
  <div class="definition">
    <h2>定义</h2>
    <p itemprop="description">在直角三角形中，两条直角边的平方和等于斜边的平方。</p>
  </div>

  <!-- 公式 (结构化展示) -->
  <div class="formula">
    <h2>公式</h2>
    <p>a² + b² = c²</p>
  </div>

  <!-- 例题 (FAQ 形式) -->
  <div class="examples">
    <h2>典型例题</h2>
    <div class="example">
      <h3>例题 1</h3>
      <p class="question">已知直角三角形两直角边为3和4，求斜边。</p>
      <details>
        <summary>查看答案</summary>
        <p>c = √(3² + 4²) = √(9+16) = √25 = 5</p>
      </details>
    </div>
  </div>

  <!-- 关联知识 (知识图谱连接) -->
  <div class="related">
    <h2>相关知识</h2>
    <a href="/explain?knowledge=151">三角函数</a>
    <a href="/explain?knowledge=152">余弦定理</a>
  </div>
</article>
```

#### 9.2.3 AI 搜索友好内容模式

| 内容模式 | 示例 | 被引用场景 |
|---------|------|----------|
| 定义 + 解释 | “勾股定理是...，它表示...” | “什么是勾股定理？” |
| 步骤列表 | “第一步...第二步...第三步...” | “如何求解二次函数顶点？” |
| 对比表格 | “顶点式 vs 一般式 vs 交点式” | “二次函数有哪些表示方法？” |
| 常见错误 | “常见错误：忘记开方、混淆直角边和斜边” | “勾股定理使用时要注意什么？” |
| 实际案例 | “应用：建筑测量中利用勾股定理...” | “勾股定理在实际生活中有什么用？” |
| 关联导航 | “学习勾股定理前，建议先掌握...” | “学习勾股定理需要什么基础？” |

#### 9.2.4 技术实现要点

| 要点 | 实现 | 说明 |
|------|------|------|
| 服务端渲染 | 服务端直出完整 HTML | AI 爬虫不执行 JS，必须服务端渲染 |
| 快速响应 | TTFB < 200ms，LCP < 2.5s | 搜索引擎排名因素 |
| HTTPS | 全站 HTTPS | 安全基础，搜索引擎要求 |
| 移动友好 | 响应式设计 + viewport | 移动优先索引 |
| 内部链接 | 知识点间相互链接，面包屑导航 | 帮助爬虫理解内容结构 |
| 面包屑 | `<nav aria-label="breadcrumb">` + BreadcrumbList Schema | 展示页面层级关系 |
| 页面速度 | 关键 CSS 内联，JS 延迟加载 | Core Web Vitals 达标 |
| 无重定向链 | 直接访问最终 URL | 减少爬虫预算消耗 |
