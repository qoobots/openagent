# 开发规范文档

## 项目概述

本项目为 OpenAgent 平台下的教育培训 AI 智能体子模块，基于 Spring Boot 3.x + MyBatis-Plus 构建，遵循分层架构设计。覆盖小学到成人终身学习全学段，服务学生、教师、家长、管理者等多类用户角色。

---

## 分层架构

```
Controller  →  Service  →  Mapper  →  Database
    ↓            ↓
   DTO          VO
```

| 层次 | 包名 | 职责 |
|------|------|------|
| Controller | `controller` | 接收请求，参数校验，调用 Service，返回 VO |
| Service | `service` / `service.impl` | 业务逻辑编排，事务管理 |
| Mapper | `mapper` | 数据访问，SQL 映射 |
| Entity | `entity` | 数据库表映射对象 |
| DTO | `dto` | 前端请求参数封装 |
| VO | `vo` | 返回给前端的数据封装 |
| Common | `common` | 通用类 (Result, ErrorCode, BusinessException) |
| Config | `config` | 配置类 |
| Util | `util` | 工具类 |

## 命名规范

### 包命名

- 基础包: `com.qoobot.agent.education_agent`
- 全小写，单词间用下划线分隔（与项目 artifactId 保持一致）

### 类命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 实体类 | `XxxEntity` 或直接 `Xxx` | `User`, `Course`, `Question`, `Homework`, `ExamRecord` |
| DTO | `XxxDTO` / `XxxRequest` | `ChatRequest`, `PlanCreateDTO`, `HomeworkSubmitDTO`, `QuestionGenerateDTO` |
| VO | `XxxVO` / `XxxResponse` | `CourseVO`, `ChatResponse`, `UserProfileVO`, `LearningProgressVO` |
| Controller | `XxxController` | `HealthController`, `TutorController`, `TeacherController`, `ExamController` |
| Service 接口 | `XxxService` | `AgentService`, `TutorService`, `HomeworkService` |
| Service 实现 | `XxxServiceImpl` | `AgentServiceImpl`, `TutorServiceImpl` |
| Mapper | `XxxMapper` | `CourseMapper`, `QuestionMapper`, `HomeworkMapper` |
| 配置类 | `XxxConfig` | `RedisConfig`, `WebMvcConfig` |
| 工具类 | `XxxUtil` | `RedisUtil` |
| 枚举 | `XxxEnum` / 语义化命名 | `ErrorCode` |
| 异常 | `XxxException` | `BusinessException` |

### 方法命名

| 操作 | 命名规范 | 示例 |
|------|----------|------|
| 查询单个 | `getXxx` | `getAgentStatus()` |
| 查询列表 | `listXxx` | `listCourses()` |
| 分页查询 | `pageXxx` | `pageStudents()` |
| 新增 | `createXxx` / `addXxx` | `createPlan()` |
| 修改 | `updateXxx` | `updateCourse()` |
| 删除 | `deleteXxx` / `removeXxx` | `deleteCourse()` |
| 判断 | `isXxx` / `hasXxx` | `isEnabled()` |

### 数据库字段

- 表名: 小写 + 下划线，业务前缀 `edu_`
- 字段名: 小写 + 下划线
- 布尔字段: `is_` 前缀，如 `is_deleted`
- 时间字段: `xxx_time` / `xxx_date`

## 统一响应格式

所有 Controller 方法必须返回 `Result<T>`：

```java
// 成功 - 无数据
return Result.ok();

// 成功 - 有数据
return Result.ok(data);

// 失败 - 使用错误码
return Result.fail(ErrorCode.AGENT_NOT_FOUND);

// 失败 - 自定义消息
return Result.fail("操作失败");
```

## 异常处理规范

### 业务异常

使用 `BusinessException` 抛出业务异常，由 `GlobalExceptionHandler` 统一捕获：

```java
// 使用错误码
throw new BusinessException(ErrorCode.AGENT_NOT_FOUND);

// 使用自定义消息
throw new BusinessException("课程不存在");

// 使用自定义 code + 消息
throw new BusinessException(1005, "自定义错误");
```

### 禁止事项

- 禁止在 Controller/Service 中直接 catch Exception 后吞掉
- 禁止返回裸字符串，必须使用 `Result` 包装
- 禁止在异常消息中暴露系统内部信息（如 SQL 语句、堆栈）

## 实体类规范

所有实体类继承 `BaseEntity`，自动获得：

- `id`: 雪花算法主键
- `createTime`: 插入时自动填充
- `updateTime`: 插入/更新时自动填充
- `deleted`: 逻辑删除字段

```java
@Data
@TableName("edu_course")
public class Course extends BaseEntity {
    private String courseName;
    private String courseCode;
    // ... 业务字段
}
```

## Mapper 规范

- 所有 Mapper 接口继承 `BaseMapperX<T>`
- 简单 CRUD 使用 MyBatis-Plus 内置方法
- 复杂 SQL 写在 `resources/mapper/*.xml` 中

```java
public interface CourseMapper extends BaseMapperX<Course> {
    // 自定义方法
}
```

## Redis 使用规范

- 使用 `RedisUtil` 工具类操作 Redis
- Key 命名规范: `education:模块:业务:id`，如 `education:course:detail:123`
- 必须设置过期时间，避免内存泄漏
- 缓存与数据库数据一致性由业务层保证

```java
@Resource
private RedisUtil redisUtil;

// 设置缓存（带过期时间）
redisUtil.set("education:course:detail:" + id, course, 30, TimeUnit.MINUTES);
```

## 日志规范

- 使用 Lombok `@Slf4j` 注解
- 级别使用:
  - `ERROR`: 系统异常、不可恢复错误
  - `WARN`: 业务异常、可恢复错误
  - `INFO`: 关键业务流程
  - `DEBUG`: 调试信息

```java
log.info("创建学习计划: userId={}, planName={}", userId, planName);
log.warn("业务异常: {}", e.getMessage());
log.error("系统异常: ", e);  // 异常对象作为最后一个参数
```

## API 文档规范

- 使用 SpringDoc (OpenAPI 3.0) 注解
- Controller 类使用 `@Tag` 标注分组
- 方法使用 `@Operation` 标注说明
- DTO/VO 字段使用 `@Schema` 标注描述

```java
@Tag(name = "课程管理", description = "课程增删改查")
@RestController
@RequestMapping("/api/course")
public class CourseController {

    @Operation(summary = "查询课程列表")
    @GetMapping("/list")
    public Result<List<CourseVO>> list() {
        // ...
    }
}
```

### 多学段接口规范

对于支持多学段的接口，建议在请求参数中包含 `educationStage` 字段：

```java
@Data
public class ChatRequest {
    @Schema(description = "学段 (elementary/middle/high/university/vocational/adult)")
    private String educationStage;
    
    @Schema(description = "用户问题")
    private String question;
}
```

## Git 提交规范

### Commit Message 格式

```
<type>(<scope>): <subject>
```

| type | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复 Bug |
| docs | 文档变更 |
| style | 代码格式（不影响逻辑） |
| refactor | 重构 |
| test | 测试相关 |
| chore | 构建/工具变更 |

示例：

```
feat(tutor): 添加智能对话辅导接口
fix(plan): 修复学习计划进度计算错误
docs(api): 更新 API 接口文档
feat(exam): 新增中高考模拟考接口
feat(teacher): 添加教师课件生成接口
feat(gamification): 实现成就徽章系统
feat(multimodal): 添加语音交互接口
```

### 分支策略

| 分支 | 说明 |
|------|------|
| `main` | 主分支，随时可发布 |
| `develop` | 开发分支 |
| `feature/xxx` | 功能分支 |
| `hotfix/xxx` | 紧急修复分支 |

## 新增业务模块 Checklist

开发新业务模块时，按以下顺序创建文件：

1. **Entity** - `entity/XxxEntity.java` - 定义数据模型
2. **Mapper** - `mapper/XxxMapper.java` - 数据访问接口
3. **DTO** - `dto/XxxDTO.java` - 请求参数对象
4. **VO** - `vo/XxxVO.java` - 响应数据对象
5. **Service** - `service/XxxService.java` - 业务接口
6. **ServiceImpl** - `service/impl/XxxServiceImpl.java` - 业务实现
7. **Controller** - `controller/XxxController.java` - 接口控制器
8. **Mapper XML** (可选) - `resources/mapper/XxxMapper.xml` - 复杂 SQL
